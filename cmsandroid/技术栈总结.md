# CMSAndroid项目技术栈总结

## 项目概述
CMSAndroid是一个基于Android平台的内容管理系统客户端，采用前后端分离架构，前端使用Android原生技术开发，后端基于Spring Boot构建RESTful API服务。

## 前端技术栈

### 核心技术
- **Java**：Android应用开发的主要编程语言，版本1.8
- **Android SDK**：Google提供的Android应用开发工具包，包含各种API和组件
  - compileSdk: 34
  - minSdk: 24
  - targetSdk: 34
- **Material Design**：Google推出的设计语言，用于构建美观、一致的用户界面，版本1.10.0

### UI组件
- **RecyclerView**：高效的列表视图组件，用于展示大量数据，支持多种布局和动画效果，版本1.3.2
- **CardView**：卡片式布局组件，用于创建具有阴影和圆角效果的卡片，版本1.0.0
- **SwipeRefreshLayout**：下拉刷新组件，用于实现列表的下拉刷新功能，版本1.1.0

### 网络通信
- **OkHttp**：Square公司开发的高效HTTP客户端库，用于与后端API进行通信，版本4.12.0
- **Gson**：Google开发的JSON序列化/反序列化库，用于处理网络请求和响应的数据转换，版本2.10.1
- **Java-WebSocket**：WebSocket客户端库，用于实现实时数据通信，版本1.5.3

## 后端技术栈

### 框架与API
- **Spring Boot**：Java后端开发框架，用于快速构建RESTful API服务
- **RESTful API**：采用REST架构风格设计的API接口，支持客户端与服务器之间的数据交互

### 安全认证
- **JWT**：JSON Web Token，用于实现无状态的用户认证和授权

### 数据库
- **MySQL/PostgreSQL**：关系型数据库，用于存储应用数据

## 开发工具

### IDE
- **Android Studio**：Google官方提供的Android应用开发集成开发环境，建议使用与AGP 8.13.2兼容的版本（如Android Studio Hedgehog或更高）

### 构建工具
- **Gradle**：现代化的项目构建工具，用于管理依赖、编译代码和打包应用，版本8.13
- **Android Gradle Plugin (AGP)**：Android项目的Gradle插件，版本8.13.2

### 版本控制
- **Git**：分布式版本控制系统，用于代码管理和团队协作

## 项目架构
CMSAndroid项目采用前后端分离架构，前端负责用户界面展示和交互，后端负责业务逻辑处理和数据存储。前端通过OkHttp库调用后端提供的RESTful API，使用JWT进行身份认证，通过Gson处理JSON数据。项目使用Git进行版本控制，通过Gradle进行构建管理。

## 技术特点
1. **原生开发**：采用Android原生技术开发，性能优秀，用户体验流畅
2. **模块化设计**：代码结构清晰，便于维护和扩展
3. **安全可靠**：使用JWT进行身份认证，保障数据安全
4. **高效通信**：采用OkHttp库进行网络通信，支持连接池、缓存等特性，提高通信效率
5. **现代化构建**：使用Gradle进行项目构建，支持自动化测试和持续集成

## 总结
CMSAndroid项目采用了主流的Android开发技术栈，结合现代化的后端架构和开发工具，构建了一个功能完整、性能优秀的内容管理系统客户端。该技术栈具有良好的可维护性和扩展性，能够满足项目的长期发展需求。
